# Исправление проблемы с исчезающим интерфейсом

## Проблема
При перезагрузке страницы интерфейс появляется на секунду и пропадает.

## Решение
Добавлена обработка ошибок во всех компонентах:

1. **Error Boundary** - перехватывает критические ошибки React
2. **Обработка ошибок API** - корректная обработка ошибок загрузки данных
3. **Настройка React Query** - уменьшено количество повторных попыток, отключен refetch при фокусе
4. **Отображение ошибок** - пользователь видит понятные сообщения об ошибках

## Что делать

1. **Перезагрузите страницу с очисткой кэша**: `Ctrl + F5` или `Ctrl + Shift + R`

2. **Проверьте консоль браузера** (F12 → Console):
   - Если есть ошибки, они будут отображаться
   - Проверьте, что backend запущен на http://localhost:8000

3. **Проверьте статус серверов**:
   ```powershell
   # Backend
   Invoke-WebRequest -Uri http://localhost:8000/health
   
   # Frontend
   Invoke-WebRequest -Uri http://localhost:3000
   ```

## Если проблема сохраняется

1. **Откройте консоль браузера** (F12)
2. **Проверьте вкладку Console** - там будут детали ошибки
3. **Проверьте вкладку Network** - все ли запросы к API успешны?
4. **Убедитесь, что backend запущен**:
   ```powershell
   cd D:\papaz\backend
   python -m uvicorn app.main:app --reload --port 8000
   ```

## Изменения в коде

- `frontend/src/main.tsx` - добавлен ErrorBoundary и настройки React Query
- `frontend/src/services/api.ts` - добавлена обработка ошибок axios
- `frontend/src/pages/*.tsx` - добавлена обработка ошибок во всех страницах
- `frontend/src/components/ErrorBoundary.tsx` - новый компонент для перехвата ошибок

Теперь при ошибках загрузки данных интерфейс не будет пропадать, а покажет понятное сообщение об ошибке.
